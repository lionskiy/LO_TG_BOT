# Acceptance criteria (PRD §7) — статус

## 7.1 Блок "Телеграм бот"
- [x] При первом входе поля пустые, статус "Not configured"
- [x] При заполнении токена и сохранении — автоматическая проверка подключения
- [x] Статус обновляется каждые 10 секунд (если токен заполнен)
- [x] Кнопка "Retry" немедленно проверяет соединение
- [x] Base URL подставляется по умолчанию, если пустой
- [x] Сохраняются только настройки этого блока
- [x] При успешной проверке — настройки немедленно применяются к приложению
- [x] Приложение начинает использовать нового бота без перезапуска

## 7.2 Блок "LLM"
- [x] При первом входе поля пустые, статус "Not configured"
- [x] При смене провайдера — автозаполнение Base URL и обновление списка моделей
- [x] При заполнении API key и сохранении — автоматическая проверка подключения
- [x] Статус обновляется каждые 10 секунд (если API key заполнен)
- [x] System prompt опционален, работает с placeholder
- [x] Сохраняются только настройки этого блока
- [x] При успешной проверке — настройки немедленно применяются к приложению
- [x] Приложение начинает использовать новую модель/провайдера без перезапуска

## 7.3 Применение настроек
- [x] Настройки применяются только после успешной проверки соединения
- [x] При неуспешной проверке — настройки сохранены в БД, но не применены
- [x] Приложение продолжает работать со старыми настройками при ошибке подключения новых
- [x] Toast-уведомление чётко указывает, применены настройки или только сохранены

## 7.4 Общее
- [x] Токены/ключи отображаются в маскированном виде
- [x] Валидация обязательных полей перед сохранением
- [x] Toast-уведомления при сохранении
- [x] Страница доступна только админам (при заданном ADMIN_API_KEY — заголовок X-Admin-Key)
